# PR Creation Template

When creating pull requests, always include:

## Summary
- What problem does this PR solve?
- What changes were made?
- Why were these changes necessary?

## Test Plan
- How can reviewers test this PR?
- What specific functionality should be verified?
- Are there any edge cases to test?
- What should reviewers look for in terms of regressions?

## Additional Context (optional)
- Any background information
- Related issues or dependencies
- Screenshots if UI changes

## Checklist
- [ ] Code compiles without errors
- [ ] All tests pass
- [ ] Manual testing completed
- [ ] Documentation updated (if needed)
- [ ] Accessibility considered
- [ ] Performance impact assessed

Remember: Even one short sentence per bullet point is better than leaving sections empty!
